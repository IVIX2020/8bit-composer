# 設計方針 (Design Document)

## アーキテクチャの概要
本プロジェクトは、マクロ（楽曲全体のタイムライン）とミクロ（個別のフレーズ編集）の2階層で構成されています。

## 相対度数ベースの音楽モデル (Relative Degree System)
絶対音高ではなく、以下のデータ構造で音高を保持しています。

```javascript
{
  deg: 1, // 1-indexed (スケール上の度数)
  oct: 5  // オクターブ
}
```

このモデルを採用することで、以下のメリットを得ています。
- **スケールの動的マッピング**: `Key` や `Scale` 設定が変更された際、同じ `degree` であっても、マッピングされる実際の周波数が新しいスケールに合わせて自動的に再計算されます。
- **創造的な再解釈**: 作成済みのフレーズの音の運び（＝楽曲の性格）を保ったまま、スケールを変更するだけで全く異なる雰囲気の楽曲へ「着せ替え」が可能です。

## モバイルファーストUI
画面の縦幅を最大限に確保するため、以下の設計を採用しています。
- 設定系パネルのアコーディオン（折りたたみ）化。
- 不要なコントロールはドロワー（サイドパネル）へ格納。
- タッチ操作を考慮したボタンサイズと遅延回避。
